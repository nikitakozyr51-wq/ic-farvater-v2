# ЭТАП 1 — Подготовка VPS (30 минут)

## 1.1 Узнай IP сервера

**Где:** Панель Beget (https://beget.com)
**Что искать:** Раздел **«VPS»** или **«Серверы»** → твой VPS → **IP-адрес** (например `87.236.16.123`)
**Что также записать:** root-пароль (если потерял — есть кнопка «Сбросить пароль»)

## 1.2 Подключиться к серверу через Termius

**Где:** Termius (на твоём компьютере)
**Шаги:**
1. Открой Termius
2. **Hosts** → **New Host** (или плюсик)
3. Заполни:
   - **Label:** `IC Farvater VPS`
   - **Hostname:** `87.236.16.123` (твой IP)
   - **Port:** `22`
   - **Username:** `root`
   - **Password:** твой root-пароль с почты Beget
4. **Save**
5. Кликни на запись → **Connect**
6. При первом подключении Termius спросит «Trust this host?» → **Yes**

**Проверка:** должна появиться строка вида `root@beget-vps:~#` — это значит ты внутри сервера. Дальше все команды копировать в эту консоль.

## 1.3 Базовая защита сервера

**Где:** SSH-сессия в Termius (которую открыла в 1.2)
**Что делать:** копировать команды по одной, нажимать Enter, ждать пока выполнится.

```bash
# Обновить пакеты ОС (займёт 2-3 минуты)
apt update && apt upgrade -y

# Поставить полезные утилиты
apt install -y curl wget ufw fail2ban htop nano git

# Настроить firewall — открыть только нужные порты
ufw default deny incoming
ufw default allow outgoing
ufw allow OpenSSH
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 3000/tcp
ufw --force enable

# Проверить
ufw status
```

**Проверка:** в выводе `ufw status` должны быть строки про порты 22 (ssh), 80, 443, 3000.

## 1.4 Установить Dokploy

**Где:** та же SSH-сессия
**Команда:**

```bash
curl -sSL https://dokploy.com/install.sh | sh
```

**Что произойдёт:** скрипт скачает Docker, Postgres (для Dokploy), Traefik (proxy) и запустит панель. Займёт 3-5 минут.

**Когда закончится** увидишь:
```
Dokploy is ready at http://YOUR_IP:3000
```

## 1.5 Войти в Dokploy

**Где:** твой браузер
**URL:** `http://87.236.16.123:3000` (подставь свой IP)
**Шаги:**
1. Откроется форма регистрации **первого** администратора
2. Заполни:
   - Email: твой
   - Password: придумай сложный, **запиши**
3. **Sign Up**
4. Войдёшь в панель Dokploy

✅ **ЭТАП 1 ЗАКОНЧЕН**

# ЭТАП 2 — Деплой сайта через Dokploy (15 минут)

1. **Create Application** → выбери **Docker**
2. Name: `frontend`

   - Type: Git

   - Repository: `nikitakozyr51-wq/ic-farvater-v2`
   - Branch: `master`
   - Build path: `/`

**Где:** Dokploy → application `frontend` → раздел **Domains**

**Шаги:**

4. Этапы (увидишь в логах):
   - `Cloning repository...`
   - `Building Docker image...`
   - `Pushing to registry...`
   - `Starting container...`
   - `Healthcheck passing ✓`
5. Статус должен стать зелёный **Running**

## 2.6 Проверить что сайт работает

**Где:** браузер
**URL:** `https://ic-farvater.ru/`
**Что должно быть:**
- Сайт открывается
- 🔒 зелёный замок (SSL работает)
- Главная страница, hero, разделы

**Шаги:**

2. Скопируй **Webhook URL** который покажет Dokploy

5. **Payload URL:** вставь URL из Dokploy
6. **Content type:** `application/json`

8. **Active:** ✓

Теперь любая правка в коде → `git push` → автоматическая пересборка сайта за 1-2 минуты.

## 3.4 Сообщить мне SMTP-параметры

- `SMTP_PORT=465`

# ЭТАП 4 — Yandex.Metrica (20 минут)

## 4.1 Зачем это нужно

Yandex.Metrica — бесплатная статистика посещений:
- Сколько человек заходит в день
- Откуда (Yandex / Google / соцсети / прямые заходы)
- На каких страницах больше сидят
- Что нажимают (карта кликов)
- Запись действий пользователей (Вебвизор — буквально видео экрана анонимного посетителя)

Для B2B сайта Metrika нужна, чтобы потом анализировать какие категории смотрят чаще, откуда приходят заявки, итд.

## 4.2 Зарегистрировать счётчик

**Где:** https://metrika.yandex.ru/list
**Шаги:**
1. **Войти** через свой Яндекс-аккаунт (тот же что для почты Yandex, или создать новый)
2. **Добавить счётчик** (большая кнопка)
3. Заполнить:
   - **Имя счётчика:** `IC Фарватер`
   - **Адрес сайта:** `https://ic-farvater.ru/`
   - **Часовой пояс:** GMT+3 Москва
4. Принять условия → **Создать счётчик**

## 4.3 Включить нужные функции

**Где:** Metrika → твой счётчик → **Настройка**
**Включи галки:**
- ☑ **Вебвизор** (запись действий)
- ☑ **Карта скроллинга**
- ☑ **Карта кликов**
- ☑ **Точный показатель отказов**

**Сохрани.**

## 4.4 Записать Counter ID

**Где:** Metrika → твой счётчик → главная страница счётчика
**Что искать:** 8-значное число вверху страницы (например `99876543`)
**Сохрани его** — понадобится через 2 минуты.

## 4.5 Подтвердить владение сайтом

**Где:** Metrika → Настройка → раздел про код счётчика
**Шаги:**
1. Yandex покажет тебе HTML-снippet кода
2. Тебе нужно его **НЕ копировать** — у нас уже всё подключено в HTML
3. Вместо этого выбери способ подтверждения **«Через мета-тег»** (если такая опция есть)
4. Yandex даст строку вида `<meta name="yandex-verification" content="abcdef123456">`
5. **Скопируй её**

## 4.6 Вписать counter ID + verification в HTML

**Где:** локально на твоём компьютере, в коде проекта
**Шаги:**
1. Открой `D:\site-v2\index.html` в редакторе (VS Code или Notepad++)
2. Найди строку:
   ```html
   <meta name="yandex-counter" content="">
   ```
3. Замени на:
   ```html
   <meta name="yandex-counter" content="99876543">
   ```
   (подставь свой 8-значный ID)
4. Чуть ниже добавь yandex-verification (из шага 4.5):
   ```html
   <meta name="yandex-verification" content="abcdef123456">
   ```
5. **Сохрани файл**
6. Открой следующий файл `D:\site-v2\pages\products.html` → повтори шаги 2-4 (только counter, не нужно yandex-verification на inner pages)
7. Так же для всех 7 HTML файлов:
   - `D:\site-v2\index.html` (counter + verification)
   - `D:\site-v2\pages\products.html`
   - `D:\site-v2\pages\product-detail.html`
   - `D:\site-v2\pages\about.html`
   - `D:\site-v2\pages\contacts.html`
   - `D:\site-v2\pages\privacy-policy.html`
   - `D:\site-v2\pages\consent.html`

**Подсказка:** если умеешь PowerShell — могу написать скрипт который сам вставит counter ID везде. Просто скажи.

## 4.7 Закоммитить и запушить

**Где:** PowerShell в папке `D:\site-v2`
**Команды:**

```powershell
cd D:\site-v2
git add -A
git commit -m "metrika: insert counter ID + verification"
git push origin master
```

Через 1-2 минуты Dokploy автоматически пересоберёт сайт (потому что в 2.7 включила autoDeploy).

## 4.8 Подтвердить в Yandex.Metrica

**Где:** https://metrika.yandex.ru/list → твой счётчик
**Шаги:**
1. **Проверить установку**
2. Yandex откроет твой сайт, поищет meta-tag, подтвердит
3. Статус счётчика станет **Активен** (зелёная точка)

## 4.9 Проверить что данные собираются

**Где:** Metrika → твой счётчик → **Сводка**
**Шаги:**
1. Открой свой сайт https://ic-farvater.ru/ в другой вкладке (или попроси кого-то)
2. Покликай по разделам
3. Через 5-10 минут вернись в Metrika → должна показываться твоя сессия

# ЭТАП 8 — Strapi админка (после одобрения V2)

Только когда заказчик скажет «ОК, делаем».

**Цель:** заказчик сам редактирует каталог и тексты страниц через веб-панель, не теребя меня.

**Время:** 1-2 рабочих дня.

**Что произойдёт:**
1. На том же VPS поднимем ещё одно приложение через Dokploy — Strapi CMS
2. + база данных Postgres
3. Привяжем subdomain `cms.ic-farvater.ru`
4. Перенесём данные из текущих JS-файлов в Strapi
5. Настроим автоматическую пересборку сайта при изменениях в Strapi

**Подробный план:** `STRAPI-ROADMAP.md` в репозитории.

**Ответы на типичные вопросы про Strapi:**

| Вопрос | Ответ |
|---|---|
| Что такое Strapi? | Это «CMS как WordPress, но для современных сайтов». Веб-панель админки + база данных |
| Зачем не WordPress? | WP сильно завязан на свою тему/плагины. Strapi только данные хранит, фронт остаётся наш — быстрый и кастомный |
| Сложно ли заказчику? | Нет — Strapi имеет интуитивный UI. Я сделаю короткое видео-руководство |
| Что если Strapi сломается? | Сайт продолжит работать (данные генерируются заранее в JS-файлы). Strapi только для редактирования |
| Будут ли расходы? | Только VPS-хостинг (который уже куплен). Сам Strapi бесплатный |
| Сколько данных можно? | На нашем VPS (5GB диска) — десятки тысяч товаров + картинок |
